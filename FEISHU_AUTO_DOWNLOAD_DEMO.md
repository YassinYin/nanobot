
### 2️⃣ 进阶：图片生成

```
👤 用户: [发送一张食物图片]
```

**原理**: 自动下载 → Vision 识别 → 工具调用 → 图片生成

```
👤 用户: [发送图片A - 你的照片]
```

## 🎯 关键要点

### ✅ 自动化

- **无需**手动下载
- **无需**提供路径
- **无需**额外配置

### 🔄 工作流程

```
发送图片 → 自动下载 → Agent 可用
```

### 💡 Agent 视角

Agent 会同时获得：
1. **Vision Input**: Base64 编码的图片（用于"看到"图片）
2. **本地路径**: 文本形式的路径（用于调用工具）

例如，Agent 收到的消息：
```
[Image downloaded to: /Users/xxx/.nanobot/media/image_20260301_152030.jpg]
```

同时图片的视觉内容通过 Vision API 传递给 LLM。

### 🎨 可以做什么

| 功能 | 示例 | 原理 |
|------|------|------|
| 识别 | "这是什么？" | Vision |
| 分析 | "分析这张图片的构图" | Vision |
| 对比 | "对比这两张图片" | Vision + 多图 |
| 生成 | "生成卡通版本" | Vision + Image-to-Image |
| 编辑 | "把背景改成海边" | Vision + Image-to-Image |
| 风格迁移 | "采用梵高的风格" | Vision + Image-to-Image |
